gr.Tab("RAG Comparison") · Compare RAGs button

⚙️ App Engine · app.py + backends/
Orchestration
🎛️
Gradio Orchestrator
app.py · run_comparison()
asyncio.gather() · 3 backends in parallel · _safe_query() error isolation
Receives a question and immediately sends it to all three search engines at the same time. If one fails, the other two keep going.
orchestrator
RAG Backends
📊
Vector RAG
backends/vector.py · retrieve() + query()
pgvector cosine · TOP_K=5 · IPv4 forced via socket.AF_INET
Finds the five document sections whose meaning is most similar to the question, measured mathematically.
TOP_K=5
🕸️
Graph RAG
backends/graph/__init__.py · route→retrieve→generate
3 modes: hybrid / entity / community · Claude routes via tool use · TOP_K=3
Classifies the question type, then follows concept relationships in a knowledge graph to surface connected facts.
3 modes
🧩
Cognee
backends/cognee_backend.py · retrieve() + query()
cognee.search(CHUNKS) · LanceDB + SQLite · TOP_K=10
Uses an automatically built hybrid index. The graph structure was created at setup time and is opaque — no manual configuration.
TOP_K=10
Shared Layer
🔢
Embeddings Singleton
backends/shared/embeddings.py · embed()
fastembed · BAAI/bge-small-en-v1.5 · 384 dims · lazy init
One shared math model converts text to numbers for all three search engines. Loaded once, reused across all queries for fairness.
shared
✍️
Answer Generation
backends/shared/claude.py · generate_answer()
claude-sonnet-4-6 · max_tokens=1024 · identical system prompt for all 3
All three search engines use the same AI writer with identical instructions. The only difference is what each engine found to write about.
shared
⚖️
RAG Evaluator
app.py · _rag_comparison_summary()
claude-sonnet-4-6 · max_tokens=1500 · 5-criterion scored table
Claude reads all three answers together and scores them on five points, then names the winner and explains why.
judge
Model Comparison Handler
🎛️
Gradio Orchestrator
app.py · run_model_comparison(question, backend)
dropdown → retrieve() once → asyncio.gather() on both models
Takes a question and the user's chosen search engine, retrieves facts once, then sends the same facts to both AI models simultaneously.
orchestrator
🔍
Selected Backend
vector_retrieve / graph_retrieve / cognee_retrieve
retrieve() returns (context_str, summary_str) · no generation call
Gathers the relevant document sections and returns them as plain text. No AI writes an answer yet — both models get this same text.
retrieve only

Claude Sonnet 4.6
backends/shared/claude.py · generate_answer()

Anthropic
🧠
Gemma 4 26B
backends/shared/gemma.py · generate_answer_gemma()
google-genai SDK · asyncio.to_thread() · Vertex AI MaaS · ADC auth
Google's Gemma 4 reads the same facts and writes its own answer. Runs at the same time as Claude.
Vertex AI
⚖️
Model Evaluator
app.py · _model_comparison_summary()
claude-sonnet-4-6 · max_tokens=1000 · Accuracy / Completeness / Clarity / Context Usage / No Hallucinations
Claude grades both answers on five criteria and names the better model for this question and context.
judge
One-Time Ingest Scripts · ingest/
📥
Vector Ingest
ingest/vector_ingest.py
PyMuPDF · RecursiveCharacterTextSplitter(500, 50) · 160 chunks · HNSW index on vector(384)
Reads all 15 policy documents, breaks them into overlapping sections, and saves mathematical fingerprints of each section to the cloud database.
160 chunks
🕸️
Graph Ingest
ingest/graph_ingest.py · 8 steps
Claude tool use (extract_graph) · Semaphore(5) · Louvain community detection · 955 entities · 855 relationships · 135 communities
Claude reads every document section and identifies key concepts and their relationships, building a web of connected knowledge across all 15 documents.
955 entities
🧩
Cognee Ingest
ingest/cognee_ingest.py
cognee.add() per PDF · cognee.cognify() once · LLM_PROVIDER=anthropic · VECTOR_DB_PROVIDER=lancedb
Feeds all 15 documents to Cognee, which automatically builds both a text search index and a knowledge graph in a single step.
15 PDFs

Starts the app automatically when the server boots and restarts it if it crashes. Connects Cognee's database files to the container so they survive restarts.
Restart=always
