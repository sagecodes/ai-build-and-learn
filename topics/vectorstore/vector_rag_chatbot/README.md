# Everstorm RAG Chatbot

A production RAG (Retrieval-Augmented Generation) chatbot that answers customer support questions from a corpus of PDF documents. Semantic search runs against a pgvector database; compute is orchestrated by [Union](https://union.ai); answers are generated by Claude.

## Architecture

```
User question
     │
     ▼
Gradio UI  ──flyte.run()──▶  Union cluster
                                   │
                          ┌────────┴────────┐
                          ▼                 ▼
                    retrieve_task     (after retrieve)
                    ┌──────────┐      generate_task
                    │ gte-small│      ┌──────────────┐
                    │ embed    │      │ Claude Sonnet │
                    │ query    │      │ RAG answer    │
                    └────┬─────┘      └──────┬───────┘
                         │                   │
                    pgvector               answer +
                    cosine                 sources
                    search
```

**Ingest pipeline** (one-time, per document corpus):

```
PDFs uploaded via UI
     │
     ├── load_and_chunk_task (parallel, one per PDF)
     │       PyMuPDF → text extraction
     │       RecursiveCharacterTextSplitter → chunks
     │
     └── embed_and_index_task
             gte-small → 384-dim vectors
             pgvector HNSW index → upsert
```

**Query pipeline** (per user question):

```
retrieve_task  →  embed query → cosine search → top-k chunks
generate_task  →  Claude Sonnet (RAG prompt) → answer
```

## Stack

| Layer | Technology |
|---|---|
| UI | Gradio 6, deployed to Union as an App |
| Orchestration | Union / Flyte 2 |
| Embedding model | `thenlper/gte-small` (384-dim, runs in task pod) |
| Vector store | Supabase (managed Postgres) + pgvector extension (HNSW index, cosine distance) |
| LLM | Claude Sonnet (`claude-sonnet-4-6`) via Anthropic API |
| PDF parsing | PyMuPDF |
| Text splitting | LangChain `RecursiveCharacterTextSplitter` |
| Container registry | Docker Hub (images built via GitHub Actions) |

## Prerequisites

- [Union account](https://union.ai) with a project created
- [Supabase account](https://supabase.com) (free tier, managed Postgres with pgvector)
- [Anthropic API key](https://console.anthropic.com)

## Setup

### 1. Clone and install

```bash
git clone <your-repo>
cd topics/vectorstore/vector_rag_chatbot

python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Supabase pgvector

Create a [Supabase](https://supabase.com) project (free tier includes 500MB Postgres). Enable the pgvector extension in the Supabase SQL Editor:

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

The `document_chunks` table and HNSW index are created automatically on first ingest — no migration needed.

**Important — use the Session Pooler URL, not the direct connection URL.** The Union cluster (AWS us-east-2) has no IPv6 connectivity, and Supabase's direct connection resolves to an IPv6 address. The Session Pooler is IPv4.

Find it in your Supabase project: **Connect → Session pooler**

```
postgresql://postgres.<project-ref>:<password>@aws-1-us-west-2.pooler.supabase.com:5432/postgres
```

### 3. Configure secrets on Union

The tasks expect two secrets injected as environment variables at runtime:

```bash
union create secret ANTHROPIC_API_KEY --value "sk-ant-..."
union create secret PG_URL --value "postgresql://user:pass@host:5432/db"
```

### 4. Configure flyte.yaml

Edit `flyte.yaml` to match your Union org, project, and domain:

```yaml
admin:
  endpoint: dns:///<your-org>.hosted.unionai.cloud
  insecure: false
task:
  project: <your-project>
  domain: development
```

### 5. Configure config.py

Update the constants at the top of `config.py`:

```python
UNION_ORG      = "<your-org>"
UNION_PROJECT  = "<your-project>"
UNION_DOMAIN   = "development"
```

### 6. Build and push Docker images

The task image includes the embedding model baked in to avoid per-run downloads. The app image provides the Gradio serving environment. Both are built via GitHub Actions on Dockerfile changes.

Add `DOCKER_HUB_TOKEN` as a GitHub Actions secret (Docker Hub → Account Settings → Personal Access Tokens), then update the image names in the workflow and Dockerfiles to your Docker Hub username:

```
docker.io/<your-username>/rag-task:latest   # task image
docker.io/<your-username>/rag-app:latest    # app serving image
```

Update the image references in `config.py` and `app.py` to match.

Trigger the first build by pushing a commit that touches either Dockerfile.

### 7. Create a .env file for local development

```bash
# .env (never committed)
FLYTE_BACKEND=union          # or "local" for local task execution
ANTHROPIC_API_KEY=sk-ant-...
PG_URL=postgresql://user:pass@host:5432/db
```

## Running locally

```bash
python app.py
```

Opens the Gradio UI at `http://localhost:7860`. Tasks run on Union (if `FLYTE_BACKEND=union`) or in-process (if `FLYTE_BACKEND=local`).

## Deploying to Union

```bash
cd topics/vectorstore/vector_rag_chatbot

flyte --endpoint <your-org>.hosted.unionai.cloud \
      --org <your-org> \
      --config flyte.yaml \
      deploy app.py \
      --project <your-project> \
      --domain development \
      serving_env
```

The deploy command bundles `app.py`, `config.py`, and `workflows.py` and uploads them to Union. The app image (`rag-app:latest`) provides the Python environment. After deploy, Union returns a public URL for the hosted app.

**Redeployment:** Run the same command after any code change. No image rebuild needed unless you change a Dockerfile.

## Ingesting documents

1. Open the app (local or deployed URL)
2. Go to **Ingest Documents**
3. Set a collection name (default: `everstorm_docs`)
4. Adjust chunk size and overlap if needed
5. Upload one or more PDFs
6. Click **Run Ingest on Union**

The ingest pipeline fans out — each PDF is chunked in parallel as a separate Union task, then all chunks are embedded and upserted to pgvector in one batch. Unchanged PDFs hit a task cache and are skipped.

Sample documents are included in `data/` (16 Everstorm Outfitters support PDFs covering returns, shipping, loyalty programs, etc.).

## Chatting

1. Go to **Chat**
2. Confirm the collection name matches what you ingested
3. Type a question and press Enter

Each question dispatches a `query_pipeline` run on Union. You can monitor task progress in real time on the Union console.

## CI/CD

`.github/workflows/build-rag-task-image.yml` builds and pushes both images to Docker Hub on changes to either Dockerfile:

- `Dockerfile` → `rag-task:latest` + `rag-task:<sha>`
- `Dockerfile.app` → `rag-app:latest` + `rag-app:<sha>`

The SHA-tagged images provide a rollback path. To roll back, update the image string in `config.py` or `app.py` to a specific SHA tag and redeploy.

## Key design decisions

**Embedding runs in the task pod, not via an API.** The `gte-small` model is baked into the task Docker image so there is no per-run download and no per-token embedding cost.

**pgvector over a managed vector database.** A single Postgres instance with the pgvector extension replaces a dedicated vector database, reducing operational complexity and cost at this scale.

**Code and image are decoupled.** Union injects the Python bundle at runtime — the Docker image only provides the environment. Code changes deploy in seconds without rebuilding images.

**Task-level caching.** `load_and_chunk_task` and `retrieve_task` are cached by inputs. Re-ingesting an unchanged PDF is free. Repeated identical queries return instantly.
